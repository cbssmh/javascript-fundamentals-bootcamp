# Study Log — JavaScript Fundamentals

이 문서는 방학 동안 JavaScript를 학습하며  
개념 이해가 어떻게 변했는지를 정리한 회고 기록이다.

날짜 단위 기록이 아니라  
**학습 단계별로 느낀 점과 이해 포인트**를 중심으로 작성했다.

## JavaScript 기초 (변수 ~ 배열)

처음에는 문법을 외우는 느낌이 강했다.  
하지만 함수와 배열 메서드를 함께 사용하면서  
**“JS는 데이터를 어떻게 흘려보내는 언어인가”**를 고민하게 되었다.

특히 map / filter / reduce를 사용하면서  
반복문보다 **의도가 드러나는 코드**가 더 중요하다는 걸 느꼈다.

## 비동기 처리 (Promise / async-await)

비동기는 가장 막연했던 파트였다.  
이전에는 async/await를 그냥 편한 문법으로만 사용했지만,  
Promise의 상태 흐름을 정리하면서  
**왜 try/catch가 필요한지**가 명확해졌다.

이후 비동기 코드를 작성할 때  
실행 순서를 예측하려는 습관이 생겼다.

## DOM과 이벤트

DOM을 처음 배울 때는  
API 목록처럼 느껴졌다.

하지만  
이벤트 → 상태 변경 → 화면 갱신  
흐름으로 정리하고 나니  
**웹이 어떻게 동작하는지 한 장면으로 보이기 시작했다.**

## this와 실행 컨텍스트

this는 가장 헷갈리는 개념이었다.  
정의 시점이 아니라 **호출 방식에 따라 결정된다**는 규칙을  
예제 코드로 반복 확인하면서 이해했다.

이후 화살표 함수의 this가  
왜 필요한지 자연스럽게 연결되었다.

## 미니 프로젝트와 구조 분리

동물 앨범 프로젝트를 진행하면서  
코드가 길어질수록 읽기 힘들어지는 경험을 했다.

이 과정에서  
“기능 단위로 나누지 않으면 유지보수가 어렵다”는 걸 체감했고,  
모듈과 컴포넌트 개념이 단순 이론이 아니라  
**필요에 의해 등장한 개념**이라는 걸 이해했다.

## SPA와 최종 프로젝트

SPA 구조는 처음엔 복잡하게 느껴졌다.  
하지만 상태(state)를 기준으로  
화면이 어떻게 바뀌는지 정리하면서  
전체 구조가 보이기 시작했다.

Trip Wiki 프로젝트를 통해  
프레임워크 없이도  
SPA의 핵심 개념을 구현해볼 수 있었고,  
다음 단계로 React를 학습해야 할 이유가 명확해졌다.
